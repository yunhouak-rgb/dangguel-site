좋은 취업2026년 4월

포트폴리오가 필요한 직무와 아닌 직무

모든 직무에 포트폴리오가 필요한 것은 아닙니다. 포트폴리오가 특히 중요한 직무는 다음과 같습니다.

  • 필수: 디자인(UX/UI, 그래픽), 개발(프론트엔드, 앱), 영상·콘텐츠 크리에이터
  • 있으면 강점: 마케팅, 기획, 데이터 분석, 콘텐츠 마케터
  • 선택적: 영업, 인사, 회계, 일반 행정

영업이나 인사직에 포트폴리오를 억지로 만들 필요는 없습니다. 하지만 마케팅이나 기획이라면 포트폴리오 하나가 서류 합격률을 크게 높여줍니다.

좋은 포트폴리오의 3가지 공통점

1. 과정이 보인다

결과물만 보여주는 포트폴리오는 약합니다. 면접관이 보고 싶은 것은 어떤 문제를 인식하고, 어떻게 접근했으며, 무엇을 배웠는가입니다. 결과보다 과정을 보여주세요.

❌ "인스타그램 게시물 50개 제작"
✅ "타겟 연령대(20대 초반) 분석 → 짧은 영상 콘텐츠로 전환 → 2개월 만에 평균 도달률 8% → 12%로 개선"

2. 숫자가 있다

수치가 없는 포트폴리오는 설득력이 없습니다. 성과를 숫자로 표현하세요. 숫자가 없다면 이전과 비교한 변화라도 써야 합니다.

3. 직무와 연결된다

지원 직무와 관련 없는 프로젝트를 많이 넣는 것보다, 관련 있는 것 2~3개를 깊게 보여주는 것이 낫습니다. 양보다 질입니다.

직무별 포트폴리오 구성 가이드

마케팅 포트폴리오

포함할 것:

  • SNS 채널 운영 경험 (팔로워 수, 도달률, 인게이지먼트)
  • 캠페인 기획·실행 사례 (목표 → 전략 → 결과)
  • 콘텐츠 예시 (실제 게시물 스크린샷)
  • 데이터 분석 결과 (어떤 콘텐츠가 왜 잘 됐는지)

구성 예시: 프로젝트 배경(1p) → 타겟 분석(1p) → 콘텐츠 전략(1p) → 결과 수치(1p) → 배운 점(0.5p)

기획 포트폴리오

포함할 것:

  • 프로젝트 기획서 (목적, 배경, 실행 계획)
  • 문제 정의 → 해결 과정 → 결과
  • 이해관계자 설득 경험
  • 수치로 측정된 성과

기획 포트폴리오는 "이 사람이 문제를 어떻게 구조화하는가"를 보여주는 것이 핵심입니다. 멋진 디자인보다 논리적인 흐름이 더 중요합니다.

UX/UI 디자인 포트폴리오

포함할 것:

  • 사용자 리서치 과정 (인터뷰, 설문, 분석)
  • 와이어프레임 → 프로토타입 → 최종 디자인 과정
  • 디자인 결정 이유 설명
  • 사용성 테스트 결과 (개선 전/후 비교)

UX 포트폴리오에서 가장 흔한 실수는 예쁜 화면만 보여주는 것입니다. 왜 그렇게 디자인했는지 이유가 없으면 약한 포트폴리오입니다.

개발 포트폴리오

포함할 것:

  • GitHub 링크 (커밋 히스토리 관리 필수)
  • 프로젝트 소개 (어떤 문제를 해결했는지)
  • 기술 스택과 선택 이유
  • 어려웠던 부분과 해결 방법
  • 실제 서비스 링크 또는 시연 영상

개발 포트폴리오는 코드 품질도 중요하지만, README를 얼마나 잘 작성했는지도 봅니다. 남이 보기 쉽게 정리하는 능력이 협업 능력을 보여줍니다.

데이터 분석 포트폴리오

포함할 것:

  • 분석 목적과 가설 설정
  • 데이터 수집·전처리 과정
  • 시각화 결과물 (차트, 대시보드)
  • 인사이트 도출과 비즈니스 제안
  • 사용한 툴 (Python, SQL, Tableau 등)

포트폴리오 제작 무료 도구

  • Notion — 텍스트·이미지 혼합, 링크 공유 편리. 기획·마케팅에 적합
  • Canva — 예쁜 디자인, PDF 제작 간편. 마케팅·콘텐츠에 적합
  • GitHub Pages — 개발자용 무료 웹 포트폴리오 호스팅
  • Figma — UX/UI 디자이너 필수 도구, 공유 링크 제공
  • Behance — 디자이너 포트폴리오 플랫폼

포트폴리오 실수 TOP 5

  • ❌ 결과물만 있고 과정이 없다 — 면접관이 가장 답답해하는 포트폴리오
  • ❌ 너무 많은 프로젝트 — 10개보다 3개를 깊게 보여주는 것이 낫다
  • ❌ 숫자가 없다 — "열심히 했습니다" 수준의 표현은 신뢰를 주지 못함
  • ❌ 지원 직무와 무관한 프로젝트만 — 관련 없는 프로젝트는 오히려 마이너스
  • ❌ 업데이트를 안 한다 — 2년 전 프로젝트만 있는 포트폴리오는 준비 부족으로 보임

포트폴리오가 없을 때 지금 당장 만드는 법

마감이 한 달도 안 남았어도 포트폴리오를 만들 수 있습니다.

  • 지난 수업 프로젝트 정리 — 이미 한 것을 문서화하기
  • 공모전 출품작 — 수상 여부 무관, 과정을 보여주면 됨
  • 사이드 프로젝트 시작 — 작은 것부터. 블로그 운영, 소규모 SNS 계정, 간단한 앱
  • 공공데이터 분석 프로젝트 — 데이터 분석 직무라면 공공데이터로 분석 후 정리

포트폴리오는 완성된 결과물이 아니어도 됩니다. "진행 중인 프로젝트"도 소재가 됩니다. 중요한 것은 지금 무언가를 하고 있다는 것을 보여주는 것입니다.
